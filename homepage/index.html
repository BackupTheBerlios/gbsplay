about

gbsplay emulates the sound hardware of the Nintendo Gameboy. It is able to play the sounds from a Gameboy module dump (.GBS format) over /dev/dsp. Also included is an XMMS input plugin and a tool to display information about a .GBS file.

gbsplay is written in C and runs on a variety of platforms: Linux, FreeBSD, Cygwin and MacOS X have been reported to work so far.

download

Release versions can be found on the download page. Currently, a source .tar.gz and Debian packages are available.

contact

To contact the development team, write a mail to gbsplay-dev@lists.uguu.de. This is a public mailing list, so your mail will be seen by others (for example in the list archives).

If you find a bug, please report it via the mailing list. Current bug, task and release status can be seen in our ditz issue tracker.

development

To follow the development process of gbsplay, you can acces the current git version: Either clone it at http://www.cgarbs.de/git/gbsplay.git/ or browse it online.

You can also subscribe to the development mailinglist or track the git changelog feed either as Atom or RSS.

If you're using Debian, you can get development updates for the packages gbsplay by adding these lines to your /etc/apt/sources.list:

deb     http://www.cgarbs.de/stuff ./
deb-src http://www.cgarbs.de/stuff ./

(more information about this repository)
